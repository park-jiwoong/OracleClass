[고객센터 - 오류코드 작성]
1. 오류코드 검색 및 결과 표시  
   - 사용자가 입력한 오류코드를 검색하여 관련 결과를 출력한다.
   - 검색 결과 상단에 해당 오류코드와 간략한 설명을 표시한다.
2. 문서 및 상세 정보 출력  
   - 오류코드에 해당하는 문서(FAQ, 가이드 등)를 상세 정보로 출력한다.
   - 추가 검색 결과(예: “상세번호” 포함 내용)를 목록 형태로 출력하며, 각 항목은 클릭 시 상세 내용을 확인할 수 있도록 한다.
3. 계정 및 디바이스 관련 정보  
   - 오류와 관련된 계정 상태 및 디바이스 이용 여부를 함께 출력하여 문제 해결에 참고할 수 있도록 한다.
4. 추가 고려 사항  
   - 반응형 디자인과 접근성을 고려하여 모바일, 데스크톱 등 다양한 환경에서 일관되게 동작하도록 한다.
   - 로딩 상태 및 에러 처리(검색 실패, 결과 없음 등)를 명확하게 표시한다.


[고객센터 - 질문 검색]
1. 질문 검색 및 전체 결과 출력  
   - 사용자가 입력한 검색어를 포함한 모든 관련 질문 및 답변을 전체 목록으로 출력한다.
2. 검색 결과 페이징  
   - 검색 결과는 1번 페이지에서 최대 6개 항목을 기본으로 출력하며, 이후 페이지는 7개씩 목록 형태로 페이징 처리하여 제공한다.
3. 상세 정보 내 관련 항목 표시  
   - 개별 질문의 상세 정보 페이지에서는 추가로 관련 질문이나 답변 목록을 별도의 목록 형태로 출력한다.
4. + 버튼 인터랙션  
   - “+” 버튼 클릭 시, 해당 질문의 추가 세부 내용이나 관련 정보를 확장하여 출력한다.
5. 추가 고려 사항  
   - 검색 결과의 필터링, 정렬(예: 최신순, 인기순) 기능을 제공하며, 사용자가 쉽게 원하는 결과를 찾을 수 있도록 한다.
   - 접근성(키보드 네비게이션, 스크린 리더 지원) 및 로딩 상태, 에러 처리 기능을 포함한다.


[고객센터 - 주제 살펴보기]
1. 주제별 콘텐츠 출력
   - 고객센터 내 주요 주제(예: 결제, 계정, 기술지원 등)에 따른 관련 콘텐츠를 목록 및 카드 형태로 출력한다.
2. 상세 정보 내 관련 항목 표시
   - 각 주제의 상세 페이지에서는 관련 항목(자주 묻는 질문, 가이드, 해결 방법 등)을 목록 형태로 제공한다.
3. + 버튼 인터랙션  
   - “+” 버튼 클릭 시, 해당 주제와 관련된 추가 정보나 세부 항목을 확장하여 출력한다.
4. 추가 고려 사항  
   - 반응형 디자인 및 접근성을 고려하여 다양한 기기에서 최적의 사용자 경험을 제공한다.
   - 로딩 및 에러 처리 상태를 명확하게 표시하고, 사용자가 쉽게 탐색할 수 있도록 한다.


[고객센터 - 문의하기]
1. 유사 문제 검색 및 권장사항 제공  
   - 사용자가 문의하기 전에 검색을 통해 유사한 문제와 해결 방법을 출력하여, 자가 해결 가능 여부를 먼저 안내한다.
2. 문의 전용 전화 안내  
   - “문의 전화를 클릭” 시, 고객센터 전화번호, 문의 코드, 예상 대기시간 등 필수 정보를 명확하게 출력한다.
3. 실시간 채팅 기능  
   - 고객센터 채팅 버튼 클릭 시, 에이전트와의 실시간 채팅 창(사이드 모달 형태)을 띄워 문의에 대한 즉각적인 답변을 제공한다.
   - 채팅 내에서 FAQ, 자동응답 기능 등을 포함하여 기본적인 답변을 우선 제공하고, 추가 문의 사항은 에이전트와 연결된다.
4. 추가 고려 사항  
   - 문의하기 페이지 역시 반응형 디자인, 접근성(스크린 리더, 키보드 네비게이션) 및 로딩/에러 처리 기능을 충실히 지원한다.
   - 고객 문의 기록 저장 및 후속 피드백 기능(이메일, SMS 알림 등)도 고려할 수 있다.
